# EG Daily Brief 자동 요약 프로토타입 진행 보고

## 1. 프로젝트 개요
본 프로젝트는 Eurasia Group의 Daily Brief PDF를 기반으로,  
일간 요약을 축적하고 주간 보고서를 자동 생성하는 프로토타입을 구축하는 것을 목표로 진행했습니다.

현재 프로토타입은 아래 기능을 수행합니다.

- PDF 파일명에서 날짜를 인식하여 일간 기사 관리
- PDF 원문 기반 일간 요약 생성
- 일간 요약 결과를 JSON 및 CSV 형태로 저장
- 주차별(월~금) 실제 존재하는 기사만 활용해 주간 보고서 생성
- 동일 주제가 반복될 경우 최신 기사 기준으로 반영
- 주간 보고서와 출처 문서를 분리 저장
- Outlook 초안 또는 발송 형태로 메일 연계 가능

---

## 2. Automate 기반으로 시작하지 않은 이유
초기에는 Microsoft Outlook 및 Power Automate 기반의 무코드 자동화를 우선 검토했습니다.  
다만 본 과업은 단순 메일 전달 자동화가 아니라, PDF 원문 해석과 주간 단위 재구성이 핵심이었기 때문에 Python 기반 프로토타입으로 방향을 전환했습니다.

주요 이유는 다음과 같습니다.

- 입력이 메일 본문이 아니라 **PDF 문서 자체**임
- PDF 내부에 **텍스트 외 이미지·도표**가 포함되어 있음
- 같은 주제를 주간 단위에서 묶고, **최신 기사 기준으로 재선별**해야 함
- 모든 항목에 대해 **출처와 원문 인용을 추적**해야 함
- 제한된 시간 내에 빠르게 검증 가능한 프로토타입이 필요했음

즉, 이번 단계에서는 운영 편의성보다 **정확한 문서 처리와 빠른 구현 가능성**을 우선했습니다.

---

## 3. 현재 프로토타입 구현 범위
현재 구현된 범위는 아래와 같습니다.

### 일간 처리
- PDF 1건 단위로 텍스트 추출
- LLM을 활용한 일간 2줄 요약 생성
- 주간 재구성을 위한 구조화 데이터 생성
- JSON 저장
- CSV 누적 저장

### 주간 처리
- 월~금 범위 내 실제 존재하는 기사만 수집
- 동일 `topic_key`는 최신 기사 기준으로 대표 항목 선택
- 아래 2개 섹션으로 최종 주간 보고서 생성
  - [주요 아젠다 별 한국 기업에의 영향]
  - [국가 및 권역별 주요 동향 - 상세]

### 결과물 저장
- `weekly_report.md` : 요약 본문
- `weekly_report_reference.md` : 출처 문서
- `weekly_report.html` : 열람용 HTML
- `weekly_report_reference.html` : 출처 열람용 HTML

---

## 4. 현재 한계
현재 버전은 프로토타입이므로 아래 한계가 존재합니다.

- 각 명령어를 수동 실행해야 함
- Outlook 메일 수신함과 첨부 저장이 자동 연계되어 있지 않음
- PDF 내 이미지·도표를 실질적으로 해석하여 요약에 반영하는 기능은 미구현
- 동일 PDF를 재처리할 경우 CSV 중복 누적 가능
- HTML 결과물은 가독성 개선 수준이며, 정식 보고서 UI 수준은 아님

---

## 5. 완전 자동화를 위해 필요한 다음 단계
현재는 `process-daily`, `build-weekly`, `send-weekly`를 각각 실행해야 합니다.  
이를 완전 자동화하려면 아래 단계가 필요합니다.

1. **Outlook 메일 수신 자동화**
   - 지정 메일함 또는 공유 메일함에서 Daily Brief 수신 감지
   - 첨부 PDF를 자동 저장

2. **스케줄 자동화**
   - 평일 수신 시 일간 요약 자동 실행
   - 금요일 오후 1시 주간 보고 자동 생성 및 발송
   - Windows 작업 스케줄러 또는 서버/VM 기반 상시 실행 환경 필요

3. **예외 처리 체계**
   - PDF 파싱 실패
   - LLM 응답 실패
   - Outlook 발송 실패
   - 로그 저장 및 관리자 알림 필요

4. **운영 환경 분리**
   - 테스트용/운영용 설정 분리
   - 수신자, 발송 모드, 주차 기준 등을 설정 파일로 통제

## 6. 성능 개선 방향
향후 정확도와 완성도를 높이기 위한 개선 방향은 아래와 같습니다.

### 1) 요약 정확도 향상
- 일간/주간 프롬프트를 실제 사용 예시에 맞춰 추가 튜닝
- `topic_key` 표준화 강화로 동일 이슈 묶음 정확도 향상
- 주간 최종 결과에 대한 후처리 검수 로직 추가

### 2) 이미지·도표 활용
현재는 PDF 텍스트 중심으로 처리하고 있습니다.  
향후에는 아래 방향을 검토할 수 있습니다.

- PDF 내 차트/도표 페이지를 별도로 추출
- 멀티모달 LLM 또는 OCR/문서 분석 모델과 연계
- 수치표, 그래프 변화, 표 내 핵심 숫자를 구조화하여 주간 요약에 반영

### 3) 결과물 품질 개선
- HTML 결과를 보고서형 템플릿으로 고도화
- 핵심 아젠다, 국가별 동향, 출처를 분리한 시각적 레이아웃 구성
- 임원용 메일 본문과 첨부 보고서를 분리 설계

---

## 7. 결론
본 프로토타입은 제한된 시간 안에, Daily Brief PDF를 기반으로  
일간 축적과 주간 재구성이 가능한 자동 요약 흐름을 검증하는 데 목적이 있습니다.

현재 단계에서는 완전 자동화보다는 **정확한 문서 처리와 주간 보고서 생성 가능성 검증**에 초점을 맞췄습니다.  
향후 Outlook 연계, 스케줄링, 이미지·도표 활용, 예외 처리 체계를 보강하면 실운영 가능한 자동화 시스템으로 확장할 수 있습니다.
